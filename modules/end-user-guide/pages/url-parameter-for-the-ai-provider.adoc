= URL parameter for the AI provider

You can use the `ai-provider=` URL parameter to specify one or more AI providers to inject into the workspace.

The URL parameter accepts a comma-separated list of provider IDs:

[source,subs="+quotes,+attributes,+macros"]
----
pass:c,a,q[{prod-url}]#__<git_repository_url>__?ai-provider=__<provider_id>__
----

.Example with a single provider
`pass:c,a,q[{prod-url}]#__<git_repository_url>__?ai-provider=google/gemini`

.Example with multiple providers
`pass:c,a,q[{prod-url}]#__<git_repository_url>__?ai-provider=google/gemini,anthropic/claude,opencodeai/opencode`

NOTE: The `ai-provider=` parameter requires that the AI tool registry ConfigMap is configured by the administrator. Provider IDs not found in the registry are ignored.
