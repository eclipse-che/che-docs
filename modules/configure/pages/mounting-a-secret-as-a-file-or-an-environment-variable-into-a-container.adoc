= Inject files from Secrets and ConfigMaps into containers


Inject configuration from {platforms-name} Secrets and ConfigMaps into workspace containers as files or environment variables, so that workspaces have access to credentials, tokens, and settings without hardcoding them in devfiles.


Secrets are {platforms-name} objects that store sensitive data (usernames, passwords, authentication tokens) in an encrypted form.

Users can mount a {platforms-name} Secret that contains sensitive data or a ConfigMap that contains configuration in a {prod-short} managed containers as:

* a file
* an environment variable

The mounting process uses the standard {platforms-name} mounting mechanism, but it requires additional annotations and labeling.
